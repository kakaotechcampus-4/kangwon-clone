## 해석 — 개선 후 (2회차)

프롬프트 4블록(출처 선택·추가 vs 일정 구분·되묻지 말고 검색·폴백) + tool docstring 4종 + `search_saved_requests` 토큰 재검색을 적용한 뒤의 결과. **1회차와 동일한 하네스·50케이스·시드**라 순수 비교다.

### 베이스라인(1회차) 대비
| 카테고리 | 라우팅 1→2회 | 답변 1→2회 |
|---|---|---|
| 참고자료 검색 | 25% → **75%** | 37.5% → **75%** |
| 저장기록 검색 | 100% → 100% | 62.5% → **87.5%** |
| 과거대화 검색 | 25% → **87.5%** | 25% → **87.5%** |
| 참고자료 추가 | 14% → **85.7%** | 100% → 100% (아래 주석) |
| 폴백/복합 | 44% → **88.9%** | 66.7% → **100%** |
| 엣지-없음 | 100% → 100% | 83% → 83% (사실상 100%, 아래 주석) |
| 엣지-비검색 | 100% → 100% | 100% → 100% |
| **총계** | **54% → 90%** | **64% → 89%** |

- **참고자료 '추가'의 진짜 개선**: 1회차 답변 100%는 *착시*였다(엉뚱하게 일정으로 저장하고 "기억했다"고 답). 라우팅이 14%→86%로 오르며 이제 실제로 `add_personal_reference`에 저장한다 — 같은 100%지만 내용이 정직해짐.
- **대화 검색**이 25%→87.5%: 신호어 확장("~라고 했지/추천받은")과 docstring 트리거가 먹힘.
- **참고자료 검색**이 25%→75%: "되묻지 말고 검색" 지시로 무검색 되묻기가 크게 줄어듦.

- **E-L5-1** "작년 크리스마스 기록?" → 답변 "저장되어 있지 않습니다"는 **정답(정직한 없음)**이나, 채점 마커에 '않습니다'가 없어 오탐으로 실패 처리됨. → 실질 엣지-없음 정확도는 6/6.

### 결론
프롬프트·docstring만으로 **라우팅 54%→90%, 답변 64%→89%**. 남은 실패는 대부분 Lv4~5의 경계·모호 사례이거나(참고자료 vs 반복일정, vague 복합) 하네스 채점 한계(E-L5-1)·week1 도구 선택(S-L3-1)이라, 프롬프트만으로 짜낼 수 있는 개선은 대체로 소진에 가깝다.
